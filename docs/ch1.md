# บทที่ 1 บทนำ

## **1.1 ความเป็นมาและความสำคัญของปัญหา**

ห้องปฏิบัติการคอมพิวเตอร์ (CKLab Computer Center) ของภาควิชาคณิตศาสตร์ สถิติ และคอมพิวเตอร์ คณะวิทยาศาสตร์ มหาวิทยาลัยอุบลราชธานี ตั้งอยู่ที่อาคาร 4 ชั้น 2 เป็นแหล่งทรัพยากรสำคัญสำหรับนักศึกษาและบุคลากร เนื่องจากมีการติดตั้งซอฟต์แวร์เฉพาะทางและเครื่องมือ AI ที่ใช้สำหรับการเรียนการสอนและการวิจัย

ในอดีต ห้องปฏิบัติการแห่งนี้ใช้ **Google Forms** เป็นเครื่องมือในการบันทึกสถิติการเข้าใช้งาน ซึ่งส่งผลให้เกิดปัญหาหลายประการ ได้แก่ ผู้ใช้ต้องเสียเวลากรอกข้อมูลซ้ำซ้อน กระบวนการช้า และเสี่ยงต่อความผิดพลาดของข้อมูล (Human Error) นอกจากนี้ยังไม่สามารถติดตามสถานะการใช้งานเครื่องคอมพิวเตอร์แบบเรียลไทม์ได้ ไม่มีระบบการจองล่วงหน้าที่มีประสิทธิภาพ และไม่สามารถวิเคราะห์รูปแบบการใช้งานเพื่อวางแผนการบริหารทรัพยากรได้

จากปัญหาดังกล่าว กลุ่มผู้พัฒนาจึงได้ริเริ่มโครงงาน **ระบบเช็คอินห้องปฏิบัติการคอมพิวเตอร์ (CKLab)** ขึ้น โดยใช้แนวคิด **"นั่งที่ไหน เช็คอินที่นั่น"** ผู้ใช้เพียงกรอกรหัส UBU WiFi ระบบจะดึงข้อมูลจาก API ของมหาวิทยาลัยโดยอัตโนมัติ ช่วยลดขั้นตอนและเวลา ทำให้ข้อมูลมีความถูกต้องแม่นยำ และช่วยให้ผู้ดูแลระบบตรวจสอบสถานะการใช้งานได้แบบ Real-time

## **1.2 วัตถุประสงค์ของโครงงาน**

1. เพื่อพัฒนาระบบเช็คอินที่เชื่อมต่อข้อมูลอัตโนมัติ (API Integration) ลดภาระการกรอกข้อมูลของผู้ใช้งาน
2. เพื่อเพิ่มประสิทธิภาพการจัดสรรทรัพยากรคอมพิวเตอร์ให้ตรงกับวัตถุประสงค์ (Resource Allocation)
3. เพื่อสร้างเครื่องมือบริหารจัดการศูนย์กลาง (Centralized Management) ที่ช่วยลดภาระงานของผู้ดูแลระบบ
4. เพื่อจัดเก็บและวิเคราะห์ข้อมูลการใช้งานเชิงลึก (Data Analytics) สำหรับการวางแผนในอนาคต

## **1.3 ขอบเขตของโครงงาน**

### 1.3.1 ฟังก์ชันที่พัฒนา

ระบบ CKLab ประกอบด้วยฟังก์ชันหลัก ดังนี้

1. **ระบบ Kiosk (หน้าจอสาธารณะ)** — การ Check-in เข้าใช้งานด้วยรหัส UBU WiFi หรือในโหมด Guest, การแสดงสถานะเซสชันพร้อม Timer, การ Check-out ออกจากระบบ และการให้คะแนนความพึงพอใจ (Feedback) หลังใช้งาน
2. **ระบบ Monitor Dashboard** — การแสดงสถานะเครื่องคอมพิวเตอร์ทุกเครื่องแบบเรียลไทม์ (ว่าง / ใช้งาน / จอง / แจ้งซ่อม) และการ Check-in / Force Logout แทนผู้ใช้สำหรับผู้ดูแลระบบ
3. **ระบบ Booking** — การจองเครื่องคอมพิวเตอร์ล่วงหน้า, การนำเข้าข้อมูลการจองจากไฟล์ CSV และการแสดงประวัติการจอง
4. **ระบบจัดการเครื่องคอมพิวเตอร์ (Manage PC)** — การเพิ่ม แก้ไข ลบข้อมูลเครื่องคอมพิวเตอร์ และการกำหนดประเภทเครื่อง (AI Station / General)
5. **ระบบจัดการซอฟต์แวร์ (Manage AI & Software)** — การจัดการรายการซอฟต์แวร์และเครื่องมือ AI พร้อมการผูกซอฟต์แวร์กับเครื่องคอมพิวเตอร์แต่ละเครื่อง
6. **ระบบรายงาน (Reports & Analytics)** — การกรองและแสดงข้อมูลสถิติการใช้งาน การส่งออกข้อมูลในรูปแบบ CSV และการแสดงกราฟวิเคราะห์
7. **ระบบตั้งค่า (System Config)** — การกำหนดค่าพื้นฐานของระบบ เช่น ชื่อห้อง ประกาศ สถานะการเปิด/ปิดบริการ และการจัดการบัญชีผู้ดูแลระบบ

### 1.3.2 กลุ่มผู้ใช้งาน

| กลุ่มผู้ใช้ | รายละเอียด | การเข้าถึงระบบ |
| :--- | :--- | :--- |
| **สมาชิกภายใน (Internal User)** | นักศึกษาและบุคลากรมหาวิทยาลัยอุบลราชธานี | ผ่านหน้าจอ Kiosk โดยกรอกรหัส UBU WiFi |
| **บุคคลภายนอก (Guest)** | บุคคลทั่วไปที่ไม่ได้สังกัดมหาวิทยาลัย | ผ่านหน้าจอ Kiosk โดยเลือกโหมด Guest Login |
| **ผู้ดูแลระบบ (Admin)** | เจ้าหน้าที่ห้องปฏิบัติการที่มีสิทธิ์จัดการระบบ | ผ่าน Admin Portal (ต้องล็อกอิน) |

### 1.3.3 ขอบเขตที่ไม่รวม

1. ระบบไม่รองรับการชำระเงินหรือการคิดค่าใช้บริการ
2. ระบบไม่รองรับการแจ้งเตือนผ่านช่องทางภายนอก เช่น LINE Notify หรือ Email (เป็นงานต่อยอดในอนาคต)
3. ระบบไม่รองรับการยืนยันตัวตนด้วย Fingerprint หรือ RFID Card
4. ระบบไม่รองรับการบริหารจัดการหลายห้องปฏิบัติการพร้อมกัน
5. ระบบไม่มี Mobile Application สำหรับโทรศัพท์มือถือ

## **1.4 ประโยชน์ที่คาดว่าจะได้รับ**

1. ลดภาระผู้ใช้งาน — ไม่ต้องกรอกข้อมูลซ้ำซ้อนเหมือน Google Forms เดิม ระบบดึงข้อมูลจาก API มหาวิทยาลัยให้อัตโนมัติ
2. ข้อมูลมีความถูกต้องแม่นยำ 100% — ลดความผิดพลาดจากการกรอกข้อมูลด้วยมือ (Human Error)
3. ผู้ดูแลระบบตรวจสอบสถานะการใช้งานได้แบบ Real-time — ลดภาระในการตรวจสอบด้วยสายตา
4. ระบบการจองล่วงหน้าช่วยลดการใช้งานซ้ำซ้อน — เพิ่มความเป็นระเบียบในการจัดการทรัพยากร
5. ข้อมูลสถิติการใช้งานที่ครบถ้วนและแม่นยำ — สนับสนุนการตัดสินใจในการบริหารจัดการและวางแผนทรัพยากรในอนาคต

## **1.5 นิยามศัพท์ / คำจำกัดความ**

| คำศัพท์ | ความหมาย |
| :--- | :--- |
| **CKLab** | ชื่อย่อของห้องปฏิบัติการคอมพิวเตอร์ CKLab Computer Center อาคาร 4 ชั้น 2 |
| **Check-in** | กระบวนการยืนยันตัวตนและบันทึกเวลาเริ่มต้นการใช้งานเครื่องคอมพิวเตอร์ |
| **Check-out** | กระบวนการสิ้นสุดเซสชันการใช้งานและบันทึกเวลาออกจากระบบ |
| **Kiosk** | หน้าจอสาธารณะสำหรับผู้ใช้ทั่วไป ไม่ต้องการการล็อกอินเพื่อเข้าถึง |
| **UBU WiFi** | รหัสบัญชีเครือข่าย WiFi ของมหาวิทยาลัยอุบลราชธานี ใช้เป็นตัวยืนยันตัวตนในระบบ |
| **Session** | ช่วงเวลาการใช้งานของผู้ใช้ตั้งแต่ Check-in จนถึง Check-out |
| **Booking** | การจองเครื่องคอมพิวเตอร์ล่วงหน้าในช่วงเวลาที่กำหนด |
| **Real-time Monitor** | การแสดงผลสถานะเครื่องคอมพิวเตอร์ที่อัปเดตโดยอัตโนมัติโดยไม่ต้องรีเฟรชหน้าเว็บ |
| **Force Logout** | การสั่งตัดเซสชันการใช้งานของผู้ใช้แทน กรณีที่ผู้ใช้ลืมกด Check-out |
| **Admin Portal** | ส่วนของระบบสำหรับผู้ดูแลระบบ ต้องการการล็อกอินด้วย Username/Password |
| **Internal User** | ผู้ใช้งานภายใน ได้แก่ นักศึกษาและบุคลากรมหาวิทยาลัยอุบลราชธานี |
| **Guest** | บุคคลภายนอกที่ใช้งานโดยไม่ผ่านการยืนยันตัวตนด้วย UBU API |
| **AI Station** | เครื่องคอมพิวเตอร์ที่ติดตั้งเครื่องมือ AI เป็นพิเศษ เช่น ChatGPT Plus, Claude Pro |
| **UBU API** | Web API ของมหาวิทยาลัยอุบลราชธานีที่ใช้ยืนยันตัวตนและดึงข้อมูลผู้ใช้อัตโนมัติ |
| **CSV** | รูปแบบไฟล์ข้อมูลตาราง (Comma-Separated Values) ใช้สำหรับนำเข้าและส่งออกข้อมูล |
| **Django** | Web Framework ภาษา Python ที่ใช้ในการพัฒนาระบบ |
| **PostgreSQL** | ระบบจัดการฐานข้อมูลเชิงสัมพันธ์แบบ Open Source ที่ใช้ในโครงงานนี้ |
